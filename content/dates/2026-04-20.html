🏗️ 物理设计 HYPERHEURIST（2026-04-21）— 超图超启发式 LLM 引擎，统一驱动 VLSI 布局布线全流程优化
arXiv: 2604.15642  |  2026-04-20 收录  |  cs.AR + cs.AI  |  华南理工大学 / 香港大学
现有 EDA 超启发式方法（如 AOS/LLaMEA）依赖单一 Python 求解器，难以捕获 VLSI 物理设计问题的超图结构（net 连接多 pin）及多层次设计规则。

HYPERHEURIST 提出超图感知超启发式 LLM 引擎：① 超图编码器将网表结构映射为 LLM 可处理的图语言表示；② 双层超启发式生成——外层 LLM 生成元启发式框架，内层 LLM 迭代进化算子；③ 工具接口适配层，支持直接驱动商业 EDA 工具（Cadence / Synopsys）和开源工具（OpenROAD）。

关键结果：在 DAC 2012/ISPD 2011/ICCAD 2014 标准基准上，Placement WL 和 Routing DRC 多目标指标全面优于现有基线（含 DREAMPlace、AutoDMP）。

亮点：超图感知是区分 VLSI 问题与一般图优化的关键，这是首个将超图结构明确编入 LLM 优化框架的工作；DAC+ICCAD 双重场景验证，工业可用性强。

✅ 功能验证 Spec2Cov（2026-04-21）— LLM 驱动规范到覆盖率的闭环验证，消除 9.47% 冗余激励，覆盖率 +11.2%
arXiv: 2604.15606  |  2026-04-20 收录  |  cs.AR  |  上海交通大学 / 华为
工业级 RTL 功能验证中，覆盖率驱动验证（CDV）的核心痛点：① 传统激励生成对覆盖分析盲目，导致大量冗余仿真；② 规范文本到测试场景转换需要大量人工。

Spec2Cov 构建规范到覆盖率 LLM 闭环框架：① 规范分析智能体解析自然语言 RTL 规范，提取可量化的功能意图（接口时序、状态机转移、数据通路规约）；② 覆盖率反馈驱动激励生成——迭代分析未覆盖分支，定向生成新测试向量；③ 冗余过滤层，主动识别并剔除已高覆盖区域的冗余激励。

评估基准：OpenTitan（工业级 RISC-V SoC）、ITC99 基准集。

关键结果：覆盖率提升 +11.2%，冗余激励减少 9.47%，平均验证效率提升 2.3×。

亮点：上海交通大学 + 华为合作；SVAGenerator（ISEDA 2026，4月14日）关注 SVA 断言生成，Spec2Cov 关注覆盖率驱动验证，两者形成验证智能化的互补链路；直接面向工业级 SoC 验证场景，是近期验证方向最具工程价值的预印本之一。

🧠 LLM微调 数据高效 Verilog LLM 微调（2026-04-21）— 合成数据 + 小规模预训练，7B 参数模型达 34B 质量
arXiv: 2604.15388  |  2026-04-20 收录  |  cs.AR + cs.CL  |  Technion（以色列理工学院）
专用 Verilog LLM（VerilogEval 基准）训练面临数据稀缺困境：公开高质量 Verilog 代码远少于软件代码，大规模预训练成本高昂。

本文系统研究数据高效 Verilog LLM 训练策略：① 合成数据生成管道——利用强 LLM 自动生成多样化的 Verilog 问题-答案对，覆盖组合/时序逻辑、FSM、算术模块等类型；② 小规模持续预训练（Continued Pretraining on Verilog Corpus）在通用 LLM 上注入 Verilog 领域知识；③ 系统消融分析：数据量、数据质量、预训练策略对微调效果的独立贡献。

关键结果（VerilogEval benchmark）：7B 参数模型经数据高效训练后，在功能正确性（pass@1）上超越 34B 规模的通用基础模型；合成数据比真实数据量扩展后效果进一步显著提升。

亮点：以色列理工学院；提出的合成数据生成方法可直接应用于 RTLCoder、VerilogCoder 等现有框架；7B 超越 34B 的参数效率结论对资源受限团队（如高校、初创公司）具有重要实践价值。

🔒 RTL安全 VeriCWEty（2026-04-21）— 基于 LLM 的 RTL CWE 安全漏洞检测框架，超越 GPT-4o 基线
arXiv: 2604.15375  |  2026-04-20 收录  |  cs.CR + cs.AR  |  佛罗里达大学
硬件安全 CWE（Common Weakness Enumeration）漏洞——如信息泄露、权限提升、侧信道时序窗口——在 RTL 级别极难自动检测，现有静态分析工具依赖人工规则，覆盖率有限。

VeriCWEty 提出LLM 驱动的 RTL CWE 安全漏洞检测框架：① 将 MITRE CWE 数据库中硬件相关条目（CWE-1190、1231、1233 等）转化为结构化检测提示（Structured Detection Prompts）；② 结合代码上下文窗口扩展技术处理大型 RTL 设计；③ 双轨输出——漏洞定位（行级）+ 修复建议生成；④ 在 CWE-Bench-Hardware（首个硬件 CWE 评估数据集）上全面评估。

关键结果：VeriCWEty 在 CWE-Bench-Hardware 上超越 GPT-4o 零样本基线，在 CWE-1190（信息曝露）和 CWE-1231（不当寄存器锁定）类型上 F1 分数分别提升 +18.3% 和 +22.7%。

亮点：佛罗里达大学硬件安全团队；是近年 RTL 安全方向继 RTLSV（形式化）、TrustLLM-HW（验证）之后，专注 CWE 分类检测的新方向；为芯片安全认证流程（CC/FIPS）提供了 LLM 增强的自动化路径。
