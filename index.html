📐 RTL生成 VerilogCL（2026-04-20）— 对比学习框架增强LLM Verilog生成鲁棒性，7B模型超越所有开源/专用/商业基线
arXiv: 2604.18162  |  2026-04-20 投稿  |  cs.AR  |  作者：Yan Tan, Tong Liu, Xiangchen Meng, Yangdi Lyu
LLM 生成的 Verilog 代码常包含语法或结构错误导致编译失败或功能不正确，限制了其在硬件设计中的可靠性。VerilogCL 通过以下三项机制增强生成鲁棒性：

① 最小错误数据增强：生成正确 RTL 与最小扰动错误 RTL 的配对训练样本，让模型学习正确与错误代码的细粒度差异；
② 对比学习（Contrastive Learning）：在表示空间中学习清晰的 validity 边界，最大化正确/错误 RTL 之间的分离度；
③ 主动筛选模块：结合语义嵌入与 token 级不确定性特征，在生成过程中过滤低置信度候选代码。

关键结果：在 VerilogEval 和 RTLLM 基准上，7B 参数模型在编译成功率和功能正确性上优于所有开源、Verilog 专用、商业 LLM 基线模型。

亮点：将 NLP 领域成熟的对比学习思路引入硬件代码生成；7B 小参数模型通过数据质量取胜的范式值得关注；与 AutoVeriFix+（轨迹感知修复）和 SiliconMind-V1 形成互补的 RTL 生成鲁棒性三角格局。

🔧 RTL修复 Clover（2026-04-19）— 神经符号智能体 + 随机思维树 RTL 修复，96.8% 修复率，比纯 LLM 多覆盖 63% bug
arXiv: 2604.17288  |  2026-04-19 投稿  |  cs.AR  |  作者：Zizhang Luo 等9人
传统 APR 依赖预定义模板，纯 LLM 方法在长 RTL 代码和波形处理中存在随机性和上下文污染。Clover 将 RTL 修复建模为结构化搜索，核心创新：

① 神经符号任务分派：不同修复操作动态分派给专门 LLM 智能体或符号求解器，兼取两者优势；
② 随机思维树（Stochastic Tree-of-Thoughts）：测试时扩展机制，将主智能体上下文管理为搜索树，平衡探索/利用；
③ RTL 专用工具箱：增强智能体与调试环境（仿真器、综合工具）的深度交互能力。

关键结果：在固定时间限制内修复 96.8% 的 bug，比纯传统方法多覆盖 94%，比纯 LLM 基线多覆盖 63%，平均 pass@1 率 87.5%。

亮点：神经符号结合是 Agentic EDA 领域较少见的路线，Tree-of-Thoughts 与硬件修复任务的结合是方法论亮点；本文是 ICCAD 截稿后涌现的 RTL 修复方向重磅成果，与 AutoVeriFix+（周期轨迹感知修复）形成差异化对比。

📊 方法论 Configuration Over Selection（2026-04-18）— 超参数敏感度超过模型选择差异，同模型最佳/最差配置差 25.5%，是平均模型间差距 5 倍
arXiv: 2604.17102  |  2026-04-18 投稿  |  cs.AR + cs.AI  |  作者：Minghao Shao, Zeng Wang, Weimin Fu, Johann Knechtel, Ozgur Sinanoglu, Ramesh Karri, Muhammad Shafique
当前 LLM RTL 生成基准测试聚焦"选哪个模型"，而忽视推理时解码配置。本文在 26 个开源 LLM 和 VerilogEval/RTLLM 基准上系统研究了这个被忽视的维度：

核心发现：对三个代表性模型进行 108 种超参数配置的大规模扫描，结果显示：
① 同一 LLM 最佳与最差配置间绝对通过率差距高达 25.5%，是平均模型间差距的 5 倍；
② 不同基准间最优配置 Spearman ρ 排名接近零，即最优配置不可跨基准迁移；
③ 在默认超参数下的基准测试混淆了"模型能力"与"配置效果"，导致排名失真。

结论：充分发挥开源 LLM 在 RTL 生成上的潜力，需要架构感知和基准感知的超参数选择方法。

亮点：NYU 团队（Ramesh Karri、Ozgur Sinanoglu）出品；对领域评测体系的方法论批判，直接影响后续论文的实验设计规范；与同组的 VeriInteresting（2026-03-16）和 Synthesis-in-the-Loop（2026-03-15）评测系列形成递进关系。

🔬 表示研究 From Natural Language to Silicon（2026-04-18）— 揭示"表示瓶颈"：IR 选择比模型选择更重要，不同 IR 仿真通过率 3%→88%
arXiv: 2604.17097  |  2026-04-18 投稿  |  cs.AR  |  作者：Weimin Fu, Zeng Wang, Minghao Shao, Johann Knechtel, Ozgur Sinanoglu, Ramesh Karri, Muhammad Shafique, Xiaolong Guo
在"自然语言→硅"流程中，用户用自然语言描述电路，LLM 将其编译为硬件中间表示（IR）。本文对六种 IR（Verilog、VHDL、Chisel、Bluespec、PyMTL3、HLS C）+ 三种前沿 LLM + 202 个任务进行系统评估：

核心发现（"表示瓶颈"）：
① 不同 IR 间仿真通过率从 3% 到 88% 不等；
② 但在任一 IR 内，不同模型间差异通常 <1.25倍；
③ 在资源受限 FPGA（iCE40）上，LLM 设计 FPGA 通过率（86.5%）高于参考（68.7%），因其简单性偏差恰好适配资源限制；
④ 可访问性-能力悖论：最用户友好的 IR（HLS C/PyMTL3）导致最差 LLM 性能。

亮点：与上面 Configuration Over Selection 是同组同日发布的姊妹论文，共同构建"是配置还是 IR 更重要"的系统性研究；表示瓶颈发现对 AI for EDA 工具链的 IR 设计有直接指导意义。

🔍 验证分析 Agentic HW Verification Token分析（2026-04-17）— 量化覆盖率上限类型，专业化智能体达 95-99% 覆盖率，Token 降低 4-13 倍
arXiv: 2604.15657  |  2026-04-17 投稿  |  cs.AR  |  作者：Vihaan Patel, Vidya Chhabria, Aman Arora
基于 LLM 的编码智能体为自动化激励生成提供了有前景的方法，但现有流程未系统分析哪些覆盖率漏洞难以填补、推理时计算资源如何分配。本文提出两层智能体框架（通用 Codex + 专业化 LangGraph），并构建覆盖率上限分类法：

① 方法论受限上限：集成硬件边界、不可行边界、死代码（这些是 LLM 无法覆盖的根本性限制）；
② 推理边界：协议排序、多模块流水线预热、窄时序条件（这些可通过专业化改善）；
③ 6 类 Token 追踪：系统提示/设计理解/激励生成/覆盖率反馈/错误恢复/智能体开销，揭示专业化如何重定向 Token 分配。

关键结果：增强系统达到 95-99% 覆盖率，Token 使用量减少 4-13 倍，收敛速度提升 2-4 倍。

亮点：首个对 Agentic 验证的 Token 经济学进行系统分析的工作；覆盖率上限分类法对评估任何 LLM 驱动验证系统的极限都具有通用价值；指导未来智能体设计在"投入更多 token"与"人工升级"之间做出正确选择。

🧬 RTL生成 COEVO（2026-04-17）— 协同进化框架联合优化功能正确性 + PPA，突破现有 LLM RTL 生成"功能正确但面积差"的核心短板
arXiv: 2604.15001  |  2026-04-17 投稿，2026-04-21 收录  |  cs.AR + cs.AI
现有 LLM for RTL 生成方法（VerilogEval、RTLCoder、VerilogCoder）主要优化功能正确性（pass@k），但生成的 RTL 在综合后 PPA 表现往往显著劣于人工编写代码——功能通过≠硅后优质。

COEVO 提出协同进化（Co-Evolutionary）框架，将功能正确性与 PPA 纳入同一优化目标：
① 双目标进化机制——同时优化通过率（功能正确）和 PPA（EDA 工具综合结果），两目标相互约束引导搜索方向；
② 代码变异策略——基于 LLM 的结构感知代码变异算子（模块重组、流水线调整、资源共享），保持功能不变的同时探索 PPA 优化空间；
③ EDA 工具闭环评估——调用 Yosys / OpenROAD 实测面积/时序/功耗，将真实 PPA 数值反馈回进化迭代；
④ 多目标 Pareto 前沿维护——保留功能正确且 PPA 各维度表现多样的候选群体，避免局部收敛。

关键结果：在 VerilogEval-Human 和 RTLLM 基准上，COEVO 生成的 RTL 在功能通过率持平或优于现有 SOTA 的同时，综合面积显著减少（部分设计 -15%~-30%），时序余量改善；与纯功能优化基线相比，Pareto 最优方案覆盖率提升明显。

亮点：首个将"功能正确性"与"PPA 质量"协同优化纳入统一进化框架的 RTL 生成工作；引入真实 EDA 工具（而非代理模型）作为适应度函数，结果与工业标准直接对比；解决了 LLM 生成 RTL"能运行但面积浪费"的根本问题，是 AI for EDA 从"功能可行"迈向"硅后优质"的关键突破。

亮点：超图感知是区分 VLSI 问题与一般图优化的关键，这是首个将超图结构明确编入 LLM 优化框架的工作；DAC+ICCAD 双重场景验证，工业可用性强。
